A 1-pager is a great document to have at the start of a project, before a more detailed Product Spec. It helps everyone get on the same page and helps:

- Develop a point of view around the opportunity, concept, and problem you're solving.
- Provide a clear mini business case — explain which metric this work will move and why it matters.
- Document (high-level) expected requirements broken out by team.
- Share early insight into proposed designs.
- Get buy-in from stakeholders.

**Guidelines for deliverable**

> [!INFO]
> **Use the below template as a starting point.**

- Not all content below is required — use your best judgment on the level of information needed.
- There may also be additional categories you want to include based on the feature.

---

| **Phase** | [Discover](status:red:bold) |
| **Driver** | [Product Manager / Driver name] |
| **Contributors** | [Tag contributors — e.g. Product Design · UX · Engineering Manager · Data Science · Analytics] |


| ### Context | [One paragraph. What is this, why does it matter, what's the expected outcome.] |
| ### Problem | - What's broken today? Use bullet points.
- Each bullet: a clear statement of a user pain or business gap.
- Include data or evidence where available. |
| ### Vision | - What does the world look like when this ships?
- Paint the picture in 2–3 bullets. |
| ### Goal | - **Value for End User** — Why/how this benefits the user. Why will they care?
- **Value for [Company Name]** — How this advances the company's objectives and/or moves leading metrics. |
| ### High-Level Requirements | - What are we envisioning building? Clearly state what we're trying to build.
- How will the end user interact with this?
- Will you need support from specialist teams (e.g. Data Science, Hardware, Research, Performance, …)? |
| ### Metrics | - Primary metric this initiative moves (e.g. _"Increase D7 retention by 5%"_).
- Secondary metrics and guardrails.
- Success criteria — how do we know we won? |
| ### What's Needed to Get Started | - What you're asking from the team: alignment, approval, feedback, resourcing, etc.
- Key dependencies or blockers.
- Recommended next steps.
- **Effort Estimate:** [S/M/L/XL]
  - S = days for 1–2 SWEs (<5 days)
  - M = weeks for 1–2 SWEs (<8 weeks)
  - L = months for 1–2 SWEs (<6 months)
  - XL = More than 6 months for 1–2 SWEs |
| ### Open Questions | - [ ] Any unknowns or uncertainties to resolve before shipping?
- [ ] Is there anything an internal R&D / research team can help clarify or quantify? |
| ### Notes | - Free-form context — links, observations, or anything that doesn't fit elsewhere. |
