# Tech Plan: Add Roadmap Confluence template + create-or-update team roadmap from `/product-doc`

**Author**: Ethan Binder

**Objective**: Add a polished, generalized Roadmap template — a team-level Confluence page with Team Mission, Project Information, and quarterly Q1/Q2/Q3/Q4 tables (Initiative · Dates · Priority · Effort · Status · Notes) — and wire `/product-doc`'s Confluence path to create-or-update this doc on opt-in. Title format `<Team Name> — Roadmap`. Auto-populates each project's row from existing signals (Effort from the One Pager intake, Status inferred from local tab files); asks the user 3 batched questions for the rest (Initiative, Dates / target quarter, Priority). Idempotent — re-running for the same project updates its row in place.

## Problem Statement

`/product-doc`'s Confluence path now publishes project-level docs (One Pager, Product Spec, Experimentation Plan, OK2Ship, Canonical Doc). What's missing is a team-level artifact — a quarterly roadmap that aggregates many projects into one view. Today the team has to maintain it by hand. Adding a generic Roadmap template + opt-in create-or-update plumbing lets `/product-doc` keep the team roadmap current as projects ship and shift, without forcing every run to touch it.

## Risks

- **Initiative drift breaks idempotency.** The Initiative cell is the row identifier. If the user types a different Initiative across runs for the same project, the skill treats it as a new row and creates duplicates. Mitigation: default the Initiative to `<Product Name>` so accepting the default keeps the value stable. SKILL.md and the registry both call out the gotcha.
- **Quarter migration during in-place updates.** If a project's target quarter changes between runs (e.g. Q3 → Q4), `confluence edit` updates the row's cells in place but doesn't move the row to a different table section. The skill flags this inline so the user knows to relocate manually. A future enhancement could detect quarter changes and do a remove-then-insert.
- **Old-text matching for empty rows.** Adding a new project to a quarter requires matching one of the empty `|  |  |  |  |  |  |` rows (now 6 cells, one per column) in that quarter's table. Since multiple quarters share the same empty-row pattern, the skill builds a stable old-text by including a few lines of context (the quarter heading + table header + the empty row). If a writer manually edits the empty rows, the next add may fail — the skill warns inline.
- **No Confluence native timeline view.** The previous design included a `<!-- extension: roadmap -->` macro placeholder for Confluence's built-in visual roadmap. That's been dropped to keep the page tight; teams who want the visual timeline can insert the macro manually in the Confluence editor on the published page.
- **Team-name collision across spaces.** Two teams with the same name in different spaces would create separate roadmaps (one per space) — that's correct, since space is the natural team boundary. If two different teams share a space and both try to create `<Team Name> — Roadmap`, the second run hits the `>1 results` branch and warns the user. Mitigation: pick distinct team names or use parent pages.
